316332 Cybersecurity — Live Classroom Demos

Runnable, browser-based demonstrations of real security & privacy design flaws, using only synthetic data.

🔍 Data Minimisation & Confidentiality

An interface that looks minimal can still leak an entire raw record over the network. This demo shows how the browser's Developer Tools reveal a hidden student ID, email, and medical accommodation note behind a page that only ever displays a name.

DevTools Network tab

🔓 Broken Access Control

Authentication proves who you are; authorisation decides what you may do. This demo shows a system that gets the first right and skips the second entirely — a Member can read an Officer's record just by editing a URL.

Insecure direct object reference
What this is (and isn't). Every demo here uses a small, entirely fictional case (the "Campus Club" service) and synthetic data — no real students, no real system, no real vulnerability being exploited against anyone. Each demo illustrates a design flaw, not an attack technique, for classroom teaching in a university cybersecurity course. See each demo's own page for the full walkthrough.
