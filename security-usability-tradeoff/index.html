Security vs. Usability Trade-off Slider

An interactive simulator exploring how high-friction security policies trigger volunteer bypass behaviors, reducing actual effective security.

📊 Real-time System Metrics

Nominal Security 50% Expected security of policy rules
User Friction 50% Annoyance level felt by volunteers
Bypass Risk 10% Risk of users bypassing controls
Effective Security 45% True protection level in practice

🖥️ Volunteer Feed & Logs

💡 Classroom Walkthrough Script & Discussion

This demo maps directly to the Week 1: Shared Responsibility and Week 3: Human Behaviour discussion prompts:

  • Low Friction, Low Security: Drag all sliders to Low. Observe that User Friction drops to 0%, but nominal and effective security drop to critical levels. The logs will report incidents of unauthorized entry due to lack of standard controls.
  • Paranoid Mode (High Friction): Drag all sliders to High. Security is nominally 100%, but User Friction hits 100%. Observe that Bypass Risk jumps to 90%, and Effective Security collapses to 10%. The logs will report volunteers actively sabotaging/bypassing controls (sharing codes, sticky notes, keeping sessions alive manually) to make their shifts manageable.
  • Finding the Sweet Spot: Adjust sliders to find a balanced GRC setup (e.g. Medium controls). Notice how reducing the friction drops the bypass risk, resulting in a significantly higher Effective Security score than the paranoid policy configuration.
